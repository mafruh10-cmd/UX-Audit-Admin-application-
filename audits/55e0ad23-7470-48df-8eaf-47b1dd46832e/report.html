Accessibility
WCAG 2.2 AA
The Workflows screen provides a structured library of HR automation prompts with a clean two-column grid layout. However, it suffers from significant accessibility gaps including low-contrast secondary text, icon-only workflow thumbnails with no alternative text, and no visible keyboard focus indicators. Usability is further weakened by truncated workflow descriptions, an under-explained filter system, and a persistent upgrade upsell that competes with primary content.

Detailed Findings
Issue 01WCAG 1.1.1 · A
Workflow icons are purely decorative lightning-bolt graphics with no accessible label
📍 Circular grey icon thumbnails on every workflow card (left column and right column throughout)
High
The Problem
Every workflow card uses an identical grey lightning-bolt icon with no visible label differentiation and no discernible alt text or ARIA label. Screen reader users cannot distinguish one workflow from another via the icon, and sighted users gain no categorical meaning from the uniform symbol.
Why critical: WCAG 1.1.1 requires all non-text content that conveys information to have a text alternative. Because the icons are the primary visual anchor of each card, their lack of differentiation also increases cognitive load by failing to leverage icon-as-category signifier.
Recommendation
Assign unique ARIA labels to each icon element (e.g. aria-label='Policy workflow') or replace with categorically distinct icons per workflow type. At minimum, mark purely decorative instances with aria-hidden='true' and ensure the card title alone is the accessible name for the interactive element.
Reference: WCAG 2.2 — 1.1.1 Non-text Content (A): all non-text content must have a text alternative
Issue 02WCAG 1.4.3 · AA
Secondary metadata text ('Used 0 times · HR') has critically insufficient contrast
📍 'Used 0 times' and 'HR' tag text beneath every workflow card description
High
The Problem
The light grey metadata text rendered against the white card background appears to fall well below the 4.5:1 contrast ratio required for normal-sized text at AA level. Users with low vision or in bright ambient environments will struggle or be unable to read this information.
Why critical: WCAG 1.4.3 mandates a minimum 4.5:1 contrast ratio for normal text. This metadata, while secondary, conveys functional information (usage frequency and category tag) that informs workflow selection decisions.
Recommendation
Darken the metadata text to at least #767676 on white (meeting 4.5:1) or preferably #595959 for comfortable reading. Run all text instances through a contrast checker tool such as WebAIM Contrast Checker before release.
Reference: WCAG 2.2 — 1.4.3 Contrast Minimum (AA): normal text requires 4.5:1 contrast ratio
Issue 03WCAG 2.4.11 · AA
No visible keyboard focus indicator on interactive elements
📍 Workflow cards, filter dropdowns ('HR', 'HR agent'), 'Clear' button, and sidebar navigation items
High
The Problem
There is no visible focus ring or outline on any interactive element in the screenshot. Keyboard-only users and switch-access users cannot determine which element is currently focused, making the entire page effectively unusable without a pointer device.
Why critical: WCAG 2.4.11 (Focus Appearance, AA in WCAG 2.2) requires focus indicators to meet minimum size and contrast thresholds. Loss of focus visibility is a complete blocker for keyboard navigation — violating the principle that system status must always be perceivable.
Recommendation
Apply a high-contrast focus outline (e.g. 3px solid #005FCC offset 2px) to all interactive elements via CSS :focus-visible. Test the entire page using Tab-key navigation and verify every focusable element is clearly distinguishable.
Reference: WCAG 2.2 — 2.4.11 Focus Appearance (AA): focus indicator must have sufficient size and contrast
Issue 04
Workflow descriptions are truncated with no mechanism to preview full content
📍 Description text on all workflow cards, e.g. 'Summarize key points from an interview — I've hosted the following interview…'
High
The Problem
Every description is cut off mid-sentence with an ellipsis, giving users only partial information to decide whether a workflow meets their need. There is no tooltip, expand affordance, or preview panel to reveal the full prompt without opening the workflow entirely.
Why critical: Progressive Disclosure requires that enough information is surfaced at each decision point to allow informed action. Forcing users to open each workflow to read its full description adds unnecessary navigation steps and increases cognitive load, violating Miller's Law by fragmenting the information needed to compare options.
Recommendation
Add a hover/focus tooltip or an inline 'expand' toggle that reveals the full prompt text without leaving the list view. Alternatively, redesign the card to show 3–4 lines of description and use a 'Read more' link for anything beyond that.
Reference: Universal Principles of Design — Progressive Disclosure: show only what is needed at each step
Issue 05
Filter dropdowns use ambiguous double-filter pattern with no explanation of difference
📍 'HR' and 'HR agent' filter dropdown buttons in the top-right of the content area
Medium
The Problem
Two filter controls are shown side by side — 'HR' and 'HR agent' — with no labels or tooltips explaining what dimension each filters by (category vs. agent type vs. tag). The visual distinction between them is only their label text, which appears nearly identical in meaning to a first-time user.
Why critical: Hick's Law states that decision time increases with the number and complexity of choices. When two options appear semantically similar without contextual explanation, users experience decision paralysis and may abandon filtering altogether.
Recommendation
Add a visible label above or beside the filter group (e.g. 'Filter by:') and use distinct placeholder labels such as 'Category: HR' and 'Agent: HR agent'. Include a brief tooltip on each dropdown explaining what it filters.
Reference: Psychology of Design — Hick's Law: too many choices slows decisions
Issue 06
'Used 0 times' counter provides no value and damages perceived product quality
📍 'Used 0 times' label beneath every single workflow card on the screen
Medium
The Problem
Every workflow displays 'Used 0 times', which communicates that none of the workflows have ever been used. This creates a negative social proof signal — implying the features are untested or untrustworthy — which can deter adoption among new users.
Why critical: Social proof is a primary driver of trust in digital products. Displaying zero usage counts is the inverse of social proof (negative proof), which according to Cialdini's persuasion principles actively reduces user confidence and willingness to engage with a feature.
Recommendation
Hide the usage counter until it reaches a meaningful threshold (e.g. display only when count ≥ 1, or show 'New' badge instead). Alternatively, seed curated workflows with aggregate org-level usage data or remove the metric entirely from the card surface.
Reference: Designing User Interfaces (Malewicz) — Social proof and trust signals in UI
Issue 07
Upgrade upsell banner competes with primary content using high-contrast dark styling
📍 Bottom-left sidebar — '7 meetings and 20 messages left this month / Upgrade for unlimited use / Upgrade button'
Medium
The Problem
The black 'Upgrade' CTA button and bold limitation text in the sidebar draw visual attention away from the main workflow content. It creates a persistent anxiety-inducing message about resource scarcity that competes with the task the user is trying to perform.
Why critical: Loss aversion (Kahneman) means that scarcity messaging creates disproportionate psychological discomfort. Placing it permanently in the primary navigation sidebar — in the highest-contrast element on the page — ensures it constantly interrupts the user's task focus.
Recommendation
Relocate the upgrade prompt to a dismissible banner at the top of the content area, triggered only when the user is near the limit threshold. Reduce the visual weight by using a secondary button style and softer colours that do not compete with the active navigation.
Reference: Psych 101 (Kleinman) — Loss aversion: people are more motivated by avoiding loss than achieving gain

Search field lacks a visible label, relying solely on placeholder text
📍 Search workflows input field at the top of the content area
Medium
The Problem
The search input has no persistent visible label — only the placeholder text 'Search workflows' inside the field. Once a user begins typing, the label disappears, leaving no contextual reminder of the field's purpose, and screen readers may not reliably announce placeholder text as a label.
Why critical: WCAG 1.3.5 and 3.3.2 require that inputs have descriptive labels that persist during interaction. Placeholder-only fields also violate the principle that form context should be maintained (Don't Make Me Think — Krug) to reduce working memory burden.

Reference: WCAG 2.2 — 3.3.2 Labels or Instructions (A): inputs must have labels that persist
Issue 09
Two-column grid collapses workflow titles inconsistently — no visual rhythm or alignment
📍 'Clarify tax responsibilities for hiring a specific country' card title — left column, fourth row
Low
The Problem
This card title wraps to two lines while all surrounding titles remain single-line, causing the card body to be taller than its row partner and breaking the visual grid rhythm. This inconsistency disrupts the Gestalt principle of continuity and makes the layout feel unpolished.
Why critical: Gestalt Law of Continuity and Similarity requires that repeating elements maintain consistent visual weight. Inconsistent card heights signal lack of design polish, which the Aesthetic-Usability Effect shows directly reduces perceived trustworthiness.
Recommendation
Set a fixed card height with overflow handled by clamping the title to one line (using CSS text-overflow: ellipsis) or constrain titles to a character limit. Ensure the grid uses CSS Grid with align-items: start to maintain consistent row alignment.
Reference: Psych 101 (Kleinman) — Gestalt: Law of Similarity and Continuity

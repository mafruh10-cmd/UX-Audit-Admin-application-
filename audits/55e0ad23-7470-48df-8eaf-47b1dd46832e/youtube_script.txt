Every enterprise HR tool has a "library" screen. It is supposed to feel like a curated toolkit. This one feels like a filing cabinet that was designed by someone who has never filed anything.

Six point one out of ten. Let's get into it.

The workflow cards each have a lightning bolt icon. Every single one. Same bolt. Same grey. No label. No ARIA label. No alt text. Nothing. WCAG 2.2 criterion 1.1.1 says all non-text content needs a text alternative, and right now a screen reader user landing on this page experiences something like a museum where every painting has the same placard that just says "art." You cannot distinguish workflows. You cannot navigate meaningfully. You are just stuck in a grey lightning storm of meaninglessness.

The metadata text underneath each card — "Used zero times · HR" — is light grey on white. It looks like someone printed it in invisible ink and then felt bad and added just a little bit of ink. WCAG 2.2 criterion 1.4.3 requires a four point five to one contrast ratio for normal text. This does not meet that. Users with low vision, users outside on a phone, users who are tired — which is everyone using HR software — cannot read it. The information is technically present and functionally gone.

There are no visible focus indicators anywhere on this page. Tab through it. Nothing lights up. Nothing rings. Nothing gives any signal that you are moving through interactive elements at all. WCAG 2.2 criterion 2.4.11 exists specifically for this. Keyboard users and switch-access users need focus appearance to navigate. Without it, this page is a touchscreen app pretending to be a web product — except it is a web product and it is doing this to itself.

Every workflow description is cut off mid-sentence with an ellipsis. No tooltip. No expand toggle. No preview. Universal Principles of Design calls this a progressive disclosure failure — you reveal information in stages based on user need, and right now the stage is "never." Users are expected to commit to opening an entire workflow just to find out whether it does what the truncated first eight words suggested it might possibly do.

Two filter dropdowns sit side by side. One says "HR." One says "HR agent." Hick's Law, from Universal Principles of Design, tells us that decision time increases with the number of choices — but it gets worse when the choices are ambiguous. These two look like the same filter had a minor identity crisis. There are no labels, no tooltips, no explanation of what dimension each one controls. A first-time user has a fifty-fifty shot at guessing correctly, which is the same odds as a coin flip, which is not a UX pattern, that is a coin flip.

Here is what the redesign does. Each icon gets a unique ARIA label or a categorically distinct visual. The decorative ones get aria-hidden equals true and the card title carries the accessible name. Metadata text goes to at minimum hash 767676 on white — run it through WebAIM first, not after. Every interactive element gets a three pixel solid focus outline via CSS focus-visible, tested manually with a keyboard before anyone ships anything. Descriptions expand via a hover tooltip or an inline toggle — three to four lines visible, "Read more" for the rest. The filters get visible labels: "Category" and "Agent type," each with a tooltip, distinct placeholder text, problem solved. The "Used zero times" counter disappears until usage crosses one, replaced by a "New" badge that does not make the product look like a ghost town.

The irony of an HR automation tool that cannot be used by keyboard is that somewhere, there is a compliance workflow in that library for exactly this.

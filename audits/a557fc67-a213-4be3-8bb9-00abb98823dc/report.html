Detailed Findings
Issue 01WCAG 1.4.1 · A
Color-only data encoding in call volume chart
📍 Call volume over time line chart
High
The Problem
The chart uses a purple fill and line as the sole means of conveying data trends, with no pattern, texture, or supplementary labeling on data points. Users with color vision deficiencies cannot distinguish or interpret the data accurately.
Why critical: WCAG 1.4.1 requires that color is not used as the only visual means of conveying information. Approximately 8% of males have some form of color blindness, making this a significant barrier.
Recommendation
Add data point markers (circles, squares) along the line, provide on-hover tooltips with exact values, and include a dashed or patterned fill option. Consider adding direct value labels at key inflection points.
Reference: WCAG 2.2 — 1.4.1 Use of Color (A): information must not be conveyed by color alone
Issue 02
Call count breakdown is visually overwhelming and hard to parse
📍 Call count breakdown section with nested boxes
High
The Problem
The call count breakdown uses a complex nested box layout with over 15 data points crammed into a single view. The hierarchy is unclear—boxes of varying sizes with thin borders and small text create significant cognitive load. Users must mentally map parent-child relationships between metrics.
Why critical: Miller's Law states working memory can hold 7±2 items; this section presents far more in a non-hierarchical layout, forcing excessive cognitive processing. The signal-to-noise ratio is poor with many zero-value metrics cluttering the view.
Recommendation
Restructure using a clear tree or flow diagram with progressive disclosure—collapse zero-value branches by default. Use indentation, stronger visual hierarchy with bolder headings, and allow drill-down interaction rather than showing everything at once.
Reference: Universal Principles of Design — Signal-to-Noise Ratio: maximise relevant information, minimise clutter
Issue 03WCAG 1.4.3 · AA
Low contrast text on light backgrounds throughout data sections
📍 Call count breakdown labels such as '0 (0%) Inbound calls', '0 New calls', and various metric labels
High
The Problem
Multiple text labels in the call count breakdown appear in light gray or muted tones against white/light backgrounds. Text like '0 Unanswered', '0 Callbacks requested' appears to have insufficient contrast ratios, likely below the 4.5:1 minimum for normal-sized text.
Why critical: WCAG 1.4.3 requires a minimum contrast ratio of 4.5:1 for normal text. Low contrast text causes reading difficulty for users with low vision, aging eyes, or those viewing in bright ambient lighting.
Recommendation
Increase text color to at least #595959 on white backgrounds to meet 4.5:1 contrast ratio. For the larger '6' numeral which functions as a heading, ensure it meets at least 3:1 for large text. Audit all metric labels with a contrast checker tool.
Reference: WCAG 2.2 — 1.4.3 Contrast Minimum (AA): normal text requires 4.5:1 contrast ratio
Issue 04WCAG 2.4.7 · AA
Dropdown filters lack visible focus indicators
📍 Filter bar: 'This office', 'Past 30 days', 'Keyword', 'Moment', 'Duration', 'Internal and external'
High
The Problem
The filter pill buttons along the top do not show any visible focus indicator styling in their current state. Keyboard-only users navigating through these controls would not know which filter is currently focused, creating a significant navigation barrier.
Why critical: WCAG 2.4.7 requires that any keyboard operable interface has a visible focus indicator. Without it, keyboard users cannot orient themselves within the interface, violating fundamental accessibility requirements.
Recommendation
Add a 2px solid outline in a contrasting color (e.g., dark purple #6C47FF) with a 2px offset around each filter when focused. Ensure the focus ring has at least a 3:1 contrast ratio against the surrounding background per WCAG 2.4.11.
Reference: WCAG 2.2 — 2.4.7 Focus Visible (AA): keyboard focus indicator must be visible
Issue 05WCAG 1.4.1 · A
Legend items in chart use color-only differentiation
📍 Call count breakdown legend (top right): Total calls, Answered calls, Missed calls, Unanswered transferred calls, Outbound (connected)
Medium
The Problem
The legend uses small colored dots (various shades of purple/blue) as the only means to distinguish between five call categories. Users with color vision deficiencies may not differentiate between the similar purple hues used for 'Missed calls', 'Unanswered transferred calls', and 'Outbound (connected)'.
Why critical: WCAG 1.4.1 prohibits using color as the sole means of conveying information. The similar purple tones are particularly problematic as they may appear identical to users with tritanopia or general color deficiency.
Recommendation
Use distinct shapes (circle, square, triangle, diamond, cross) for each legend item in addition to color. Alternatively, use patterns or ensure each color has sufficient luminance difference (at least 3:1 between adjacent legend colors).
Reference: WCAG 2.2 — 1.4.1 Use of Color (A): information must not be conveyed by color alone
Issue 06
Zero-value metrics dominate the view creating noise
📍 Call count breakdown: multiple boxes showing '0 Answered', '0 Missed', '0 Abandoned', '0 Unanswered transfer', etc.
Medium
The Problem
The majority of displayed metrics show zero values, yet they occupy the same visual prominence as the meaningful data (6 Total calls, 6 Outbound calls). This creates a poor signal-to-noise ratio where the user must scan through many irrelevant zeros to find actionable insights.
Why critical: The Signal-to-Noise Ratio principle states that maximizing relevant information and minimizing clutter improves comprehension. Showing 15+ zero-value metrics at full size violates this, increasing time to insight and user frustration.
Recommendation
Collapse or de-emphasize zero-value metrics by default—show them as a muted summary line ('12 other categories: 0 calls') with an expand option. Highlight non-zero metrics with stronger visual weight to guide attention to meaningful data.
Reference: Universal Principles of Design — Signal-to-Noise Ratio: maximise relevant information, minimise clutter
Issue 07
Duplicated navigation sections in sidebar
📍 Left sidebar: 'AI Assistants', 'Reports', 'Playbooks' sections appear twice
Medium
The Problem
The left sidebar contains duplicate navigation groups—'AI Assistants', 'Reports (New)', and 'Playbooks' with sub-items appear both in the upper and lower portions of the sidebar. This creates confusion about whether these are different sections or the same content, violating consistency expectations.
Why critical: Nielsen's Heuristic #4 (Consistency and Standards) states that users should not have to wonder whether different words, situations, or actions mean the same thing. Duplication breaks the user's mental model of navigation structure.
Recommendation
Remove the duplicate navigation group. If the lower section is meant for a different context (e.g., personal vs. team), differentiate with clear section headers like 'Team Navigation' vs. 'Personal Workspace'. Ensure each item appears only once in the navigation hierarchy.
Reference: Don't Make Me Think (Krug) — Billboard Design 101: users scan, not read
Issue 08WCAG 2.5.8 · AA
Banner warning text is small and dismissal target is minimal
📍 Top yellow warning banner about external messages
Medium
The Problem
The warning banner about not being able to send external messages contains a small 'Click here' link and a tiny 'X' close button. The close button appears to be under 24×24px, making it difficult to tap on touch devices. The 'Click here' link text is also non-descriptive.
Why critical: WCAG 2.5.8 requires a minimum target size of 24×24 CSS pixels for interactive elements. Fitts' Law also indicates that smaller targets increase error rates and interaction time, particularly on mobile or touch interfaces.
Recommendation
Increase the close button to at least 44×44px touch target (24×24px minimum per WCAG). Replace 'Click here' with descriptive text like 'Learn how to register your company'. Add adequate padding around interactive elements.
Reference: WCAG 2.2 — 2.5.8 Target Size Minimum (AA): interactive targets must be at least 24x24 CSS pixels
Issue 09
Tab navigation lacks clear active state contrast
📍 Tabs: Calls, Moments, Duration, Texts
Low
The Problem
The active 'Calls' tab uses only a subtle underline and slight color change to indicate the selected state. The difference between active and inactive tabs is minimal, making it hard to quickly identify which tab is currently viewed, especially for users with low vision.
Why critical: The Aesthetic-Usability Effect aside, clear visual feedback about the current system status is essential per Nielsen's Heuristic #1 (Visibility of System Status). Insufficient differentiation between active and inactive states slows orientation.
Recommendation
Strengthen the active tab indicator with a thicker underline (3-4px), bolder font weight, and a higher-contrast color. Consider adding a subtle background color to the active tab to create a stronger selected state. Ensure the indicator has at least 3:1 contrast with the surrounding area.
Reference: Universal Principles of Design — Visibility: system status and available actions must be visible
Issue 10
Export action is visually deprioritized and easily missed
📍 'Export (.csv)' link with dropdown chevron, top right of chart section
Low
The Problem
The Export (.csv) action is styled as a subtle text link in the upper right corner of the chart area, making it easy to overlook. For a data analytics dashboard where export is a primary action, this is inappropriately deprioritized.
Why critical: Norman's Signifier principle states that visible cues must indicate where to act. Key actions should have appropriate visual affordance proportional to their importance in the user's workflow. Analytics users frequently need to export data.
Recommendation
Style the export action as a secondary button with a border, icon, and clearer affordance. Position it in a consistent action bar above or beside the chart title. Use an export/download icon alongside the text for faster recognition.
Reference: The Design of Everyday Things (Norman) — Signifier: visible cues must indicate where to act

Detailed Findings
Issue 01WCAG 1.4.1 · A
Color-only stage indicators lack text differentiation
📍 Stage indicators next to 'Meeting' and 'Prospecting' in activity timeline and right panel
High
The Problem
Stage values use small colored dots (yellow for Meeting, green for Prospecting) as the sole visual differentiator between stages. Users with color vision deficiencies cannot distinguish between stage states by color alone.
Why critical: WCAG 1.4.1 requires that color is not used as the only visual means of conveying information. Approximately 8% of males have color vision deficiency and would miss this status distinction.
Recommendation
Add a secondary visual indicator such as distinct icons or shapes for each stage (e.g., circle for Prospecting, diamond for Meeting). Alternatively, ensure the text label is always prominently paired with the dot.
Reference: WCAG 2.2 — 1.4.1 Use of Color (A): information must not be conveyed by color alone
Issue 02WCAG 1.4.3 · AA
Category tag colors have insufficient contrast with white text
📍 Categories row in Record Details: 'Retail', 'Food', 'Restaurants', 'B2C' tags
High
The Problem
The colored category tags use saturated backgrounds (green for 'Retail', orange for 'Food', teal for 'Restaurants', blue for 'B2C') with white text. Several of these combinations likely fail the 4.5:1 contrast ratio for normal-sized text, particularly the green and orange tags.
Why critical: WCAG 1.4.3 requires a minimum contrast ratio of 4.5:1 for normal text. Low-contrast text increases cognitive effort and is particularly problematic for users with low vision or in bright environments.
Recommendation
Use darker text colors on colored tag backgrounds or darken the tag backgrounds sufficiently to achieve 4.5:1 contrast. Consider using outlined/bordered tags with dark text for better readability.
Reference: WCAG 2.2 — 1.4.3 Contrast Minimum (AA): normal text requires 4.5:1 contrast ratio
Issue 03WCAG 1.4.3 · AA
Medium priority tag has insufficient color contrast
📍 'Medium' priority badge in activity timeline
High
The Problem
The yellow/gold 'Medium' priority badge uses a light background with relatively light text. This color combination may not meet the 4.5:1 minimum contrast requirement, making it difficult to read for users with low vision.
Why critical: Per WCAG 1.4.3, text must maintain sufficient contrast against its background. Yellow-on-white or yellow-toned badges are common contrast violators that disproportionately affect users with visual impairments.
Recommendation
Darken the text within the badge or use a darker badge background color. An amber/dark orange text on a light yellow background typically achieves adequate contrast while maintaining the semantic color meaning.
Reference: WCAG 2.2 — 1.4.3 Contrast Minimum (AA): normal text requires 4.5:1 contrast ratio
Issue 04WCAG 1.1.1 · A
Star rating for close confidence lacks accessible text alternative
📍 Close confidence row showing 5 yellow stars in the activity timeline
High
The Problem
The close confidence value is conveyed as a visual star rating (appears to be 3 out of 5 stars). Screen reader users and users who cannot perceive color differences between filled and unfilled stars will not understand the confidence level.
Why critical: WCAG 1.1.1 requires that all non-text content has a text alternative. Star ratings are images of information that need programmatic text equivalents to be accessible.
Recommendation
Add an aria-label or visible text like '3 out of 5' alongside the star visualization. Ensure unfilled vs filled stars differ by more than color alone (e.g., outline vs filled shape).
Reference: WCAG 2.2 — 1.1.1 Non-text Content (A): all non-text content must have a text alternative
Issue 05
High information density in activity timeline creates cognitive overload
📍 Main activity feed showing expanded change log with 8+ attribute changes
Medium
The Problem
The expanded activity entry shows 8 attribute changes simultaneously, mixing stage changes, priority, owner, estimated value, dates, and confidence ratings. This creates a dense wall of data that is difficult to scan quickly.
Why critical: Per Miller's Law, working memory can hold approximately 7±2 items. Displaying all 8 changes expanded by default exceeds this threshold and increases cognitive load, making it harder to extract key information.
Recommendation
Show only the 2-3 most important changes by default (Stage, Priority, Owner) with a 'Show N more changes' expandable section. Use visual grouping to cluster related changes together.
Reference: Universal Principles of Design — Progressive Disclosure: show only what is needed at each step
Issue 06WCAG 2.5.8 · AA
Small touch/click targets for action icons in the right panel
📍 Small icons next to Sales list item (expand and overflow menu icons at top right of Sales card)
Medium
The Problem
The expand and three-dot menu icons next to the 'Sales' list entry in the right panel appear to be very small interactive targets, likely under 24×24 CSS pixels. This makes them difficult to activate, especially on touch devices or for users with motor impairments.
Why critical: WCAG 2.5.8 (Target Size Minimum) requires interactive targets to be at least 24×24 CSS pixels. Small targets increase error rates and frustration per Fitts' Law — smaller targets require more precision and time to click.
Recommendation
Increase the clickable area of these icons to at least 24×24px (ideally 44×44px for touch). Use padding around the icon to enlarge the hit area without changing the visual size.
Reference: WCAG 2.2 — 2.5.8 Target Size Minimum (AA): interactive targets must be at least 24x24 CSS pixels
Issue 07
Truncated description text lacks full disclosure mechanism
📍 Description field in Record Details showing 'sweetgreen offers simple, seasonal, and he...'
Medium
The Problem
The company description is truncated with an ellipsis but there is no visible affordance (tooltip, expand button, or hover indication) to read the full description. Users cannot access the complete information without additional discovery.
Why critical: Per Norman's Gulf of Evaluation, users cannot tell if or how they can access the full content. The truncation provides no signifier for how to reveal the rest, violating the principle that the system should make the state of information visible.
Recommendation
Add a 'Show more' link or ensure the text expands on click. Alternatively, provide a tooltip on hover that displays the full description. Make the interactive nature of the field visually obvious.
Reference: The Design of Everyday Things (Norman) — Signifier: visible cues must indicate where to act
Issue 08
Inconsistent arrow notation between activity changes
📍 Activity timeline change entries using '→' arrows between old and new values
Low
The Problem
The change log uses arrow symbols (→) to indicate transitions, but the first Stage change shows a transition from blank to 'Meeting' while the last shows a transition from blank to 'Prospecting', creating ambiguity about whether these are sequential or independent changes.
Why critical: Per Gestalt's Law of Similarity, visually identical patterns should convey the same meaning. The identical arrow format for both a 'set to' and 'changed from/to' operation creates potential confusion about the nature of the change.
Recommendation
Differentiate between 'initial set' operations (e.g., 'Set to Meeting') and 'changed' operations (e.g., 'Prospecting → Meeting'). Use distinct visual language for creation vs. modification events.
Reference: Psych 101 (Kleinman) — Gestalt: Law of Similarity
Issue 09
Navigation breadcrumb at top is low contrast and small
📍 Top bar text '1 of 1 in Stage → Meeting'
Low
The Problem
The navigation context text '1 of 1 in Stage → Meeting' at the top uses a small font size and gray color that provides limited contrast against the white background, making it easy to overlook as a navigation aid.
Why critical: This contextual navigation information helps users understand their position within a filtered view. Low visibility undermines the Visibility of System Status heuristic, as users may not realize they're viewing a filtered subset.
Recommendation
Increase the text size slightly and use a darker gray (at least #595959 on white) to improve readability. Consider adding a subtle background or border to make this contextual element more prominent.
Reference: Designing User Interfaces (Malewicz) — Contrast and visual hierarchy
Issue 10
Dual 'Show all values' links create redundancy
📍 Right panel Record Details section and Sales list section both have 'Show all values' links
Low
The Problem
Both the Record Details section and the Sales list card include 'Show all values' links, which expand to reveal additional fields. The identical labeling doesn't differentiate which data each link reveals, potentially confusing users about what they'll see.
Why critical: Per Don't Make Me Think, identical labels for different actions force users to think about which one to click. Ambiguous labels increase the cognitive cost of interaction and reduce scanning efficiency.
Recommendation
Make the labels more specific: 'Show all record details' and 'Show all list values' respectively. This removes ambiguity and helps users quickly identify which section they want to expand.
Reference: Don't Make Me Think (Krug) — Billboard Design 101: users scan, not read

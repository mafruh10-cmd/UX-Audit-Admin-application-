# The Design of Everyday Things
## Source: Don Norman — Revised & Expanded Edition (2013)

## Key Principle for UX Auditing

DOET is the foundational text of Human-Centered Design. Use these principles to identify root causes
in UI/UX audits and cite specific concepts (e.g. 'Norman — missing signifier', 'Gulf of Evaluation').

✅ The six fundamental design principles (Affordance, Signifier, Constraint, Mapping, Feedback, Conceptual Model) must be checked in every UI audit.
✅ The Seven Stages of Action explain why users get stuck or make errors.
✅ Gulfs of Execution and Evaluation identify where design fails users.
✅ Human Error is almost always design error — never blame the user.

### The Six Fundamental Design Principles
**Core idea:** Good design requires Affordances, Signifiers, Constraints, Mappings, Feedback, and a good Conceptual Model.

### Affordance
**Core idea:** The relationship between a physical object and a person that determines how it could possibly be used.
**UX implication:** Affordances exist even if they are not visible. For designers, their visibility is critical: visible affordances provide strong clues to the operations of things. A flat plate mounted on a door affords pushing. Knobs afford turning, pushing, and pulling. Slots are for inserting things into. Balls are for throwing or bouncing. **Core idea:** Signals that communicate where the action should take place — more important than affordances for design.
**UX implication:** Signifiers communicate where the action should take place. We need both.
People need some way of understanding the product or service they wish to use, some sign of what it is for, what is happening, and what the alternative actions are. People search for clues, for any sign that might help them cope and understand. It is the sign that is important, anything that might signify meaningful information. ## Quick-Reference: DOET Principles for UI Audit

| Principle | Fail Condition | Common UI Example |
|-----------|---------------|-------------------|
| **Affordance** | Element looks clickable but isn't (or vice versa) | Flat design buttons with no visual cue |
| **Signifier** | No visible cue where to act | Icon-only buttons, unlabelled inputs |
| **Feedback** | Action gives no response | Form submit with no confirmation |
| **Mapping** | Controls don't spatially match effects | Confusing tab order, mismatched toggles |
| **Constraint** | Nothing prevents user error | No validation, irreversible destructive actions |
| **Conceptual Model** | System behaviour surprises the user | Hidden state, unclear mode changes |
| **Gulf of Execution** | User can't figure out how to do what they want | Buried features, no clear CTA |
| **Gulf of Evaluation** | User can't tell if their action worked | No loading state, no success/error message |
| **Slip (Error)** | Wrong action on correct goal | Confirm/cancel button order reversed |
| **Mistake (Error)** | Wrong mental model leads to wrong goal | Misleading labels, confusing hierarchy |
| **Forcing Function** | Dangerous action has no guard | Delete without confirmation dialog |
| **Discoverability** | Features are undiscoverable | Settings hidden behind icons, no empty state |
